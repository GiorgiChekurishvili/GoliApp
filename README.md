#  GoliApp - საფეხბურთო აპლიკაცია

**GoliApp** არის თანამედროვე Android აპლიკაცია ფეხბურთის მოყვარულთათვის. ის მომხმარებელს საშუალებას აძლევს თვალი ადევნოს მიმდინარე მატჩებს, ნახოს ლიგების ცხრილები, დაამატოს სასურველი თამაშები ფავორიტებში და მიიღოს დეტალური ინფორმაცია თითოეული მატჩის შესახებ.

##  ფუნქციონალი

*   **მატჩების სია:** დღევანდელი თამაშების ნახვა ლიგების მიხედვით.
*   **დეტალური ინფორმაცია:** მატჩის სტატუსი, ანგარიში, თარიღი და გუნდების ლოგოები.
*   **ფავორიტები:** მატჩების შენახვა ლოკალურ ბაზაში (Room), რათა ინფორმაცია ხელმისაწვდომი იყოს ინტერნეტის გარეშეც.
*   **სატურნირო ცხრილები:** სხვადასხვა ლიგების მიმდინარე მდგომარეობის (Standings) ნახვა.
*   **Swipe-to-Refresh:** მონაცემების მომენტალური განახლება ეკრანის ჩამოწევით.
*   **Live სტატუსი:** მიმდინარე მატჩების გამოყოფა სპეციალური ვიზუალური ეფექტებით.

##  ტექნოლოგიური სტეკი

აპლიკაცია აგებულია საუკეთესო პრაქტიკების და თანამედროვე ბიბლიოთეკების გამოყენებით:

*   **ენა:** Kotlin
*   **არქიტექტურა:** MVVM (Model-View-ViewModel) - მონაცემებისა და ვიზუალის სრული განცალკევებისთვის.
*   **Dependency Injection:** Hilt - ობიექტების მართვისა და ინექციისთვის.
*   **Networking:** Retrofit 2 & OkHttp - API-Football-თან კომუნიკაციისთვის.
*   **Database:** Room - ლოკალური ქეშირებისა და ფავორიტების შესანახად.
*   **UI Binding:** ViewBinding - კომპონენტებთან უსაფრთხო წვდომისთვის (პროექტში არ არის გამოყენებული `findViewById`).
*   **Navigation:** Navigation Component - ფრაგმენტებს შორის გადასვლებისა და SafeArgs-ის მართვისთვის.
*   **Image Loading:** Glide - გუნდების ლოგოების ასინქრონული ჩატვირთვისთვის.
*   **Concurrency:** Kotlin Coroutines & Flow - ასინქრონული პროცესების მართვისთვის.

##  არქიტექტურული დეტალები

პროექტი დაყოფილია რამდენიმე ფენად (Layered Architecture):
1.  **Data Layer:** მოიცავს DTO მოდელებს, Retrofit სერვისებს და Room მონაცემთა ბაზას.
2.  **Domain Layer:** შეიცავს ძირითად Match და League მოდელებს და Mapper-ებს მონაცემთა ტრანსფორმაციისთვის.
3.  **Repository Layer:** შუამავალი რგოლი, რომელიც წყვეტს მონაცემები ინტერნეტიდან წამოიღოს თუ ლოკალური ბაზიდან.
4.  **UI Layer:** ViewModel-ები, რომლებიც ამუშავებენ ლოგიკას და Fragment-ები, რომლებიც მხოლოდ ვიზუალის ჩვენებაზე არიან პასუხისმგებელნი.

##  სავალდებულო მოთხოვნების შესრულება

*   **მენიუ:** რეალიზებულია BottomNavigationView-ს საშუალებით.
*   **ლისთი:** გამოყენებულია RecyclerView მატჩებისა და ცხრილების საჩვენებლად.
*   **MVVM:** სრულად დაცულია აპლიკაციის არქიტექტურული გამართულობა.
*   **ბაზასთან კავშირი:** გამოყენებულია როგორც Retrofit (Remote), ასევე Room (Local).
*   **findViewById:** აპლიკაციაში არ არის გამოყენებული არცერთი findViewById, UI მართვა ხდება ViewBinding-ის მეშვეობით.
*   **ახალი ფუნქციონალი:** გამოყენებულია SwipeRefreshLayout დინამიური განახლებისთვის და SafeArgs ნავიგაციისთვის.

##  ინსტალაცია და გაშვება

1. დააკლონეთ რეპოზიტორი გითჰაბიდან.
2. გახსენით პროექტი Android Studio-ში.
3. დარწმუნდით, რომ გაქვთ ინტერნეტთან კავშირი API-სთან დასაკავშირებლად.
4. დააჭირეთ **Run** ღილაკს.
